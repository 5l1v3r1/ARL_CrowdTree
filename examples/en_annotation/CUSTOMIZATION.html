Customization

1.0 Servlet options

1.1 File format

The example is set up to read and write in CoNLL-U format. The code for reading in the input file(s) is specified by the following line of examples/en_annotation/annotation_configuration.argmap 
---+InstanceReader arl.nlp.parse.io.CoNLLUParseReader
Similarly, the code for writing out the annotation results in specified by this line 
--+AnnotationRecorder arl.nlp.parse.io.CoNLLUParseWriter

1.2 Punctuation filtering and attachment

Depending on your use case, it may or may not be desirable to have annotators spend their time annotating punctuation. To mark certain tokens as punctuation (NOTE: punctuation is filtered from the tree view, although it still appears in the sentence at the top of the frontend), it is possible to specify a punctuation filter. Below is the relevant line of examples/en_annotation/annotation_configuration.argmap 
--+PunctuationFilter arl.nlp.crowdling.DefaultPunctuationFilter
If punctuation is filtered from annotation, it may be desirable to attach it using a heuristic once annotation is complete. Such an attachment heuristic can be specified by editing the following line of examples/en_annotation/annotation_configuration.argmap 
--+PunctuationHeuristic arl.nlp.punctuation.DefaultPunctuationAttachmentHeuristic

1.3 Text direction

If you are working with a script that reads right-to-left instead of left-to-right (Arabic script, for example), change the following line in examples/en_annotation/annotation_configuration.argmap from 
--TextDirection ltr
to 
--TextDirection rtl

1.4 Initial tree structure

The example is configured so that all nodes will initially be attached to the root node. If you would prefer that the trees specified in the input file are not 'flattened' in this manner but are instead shown in their original state, change the following line in examples/en_annotation/annotation_configuration.argmap from 
--FlatTrees true
to 
--FlatTrees false

2.0 Client-side options

2.1 Dependency labels

2.1.1 Enabling/disabling dependency label visibility

To control whether the dependency labels are visible or not, look in client/js/tree_editor_app.js for the line with 
.setShowDep
and set it to 
.setShowDep(true)
or 
.setShowDep(false)
respectively.

2.1.2 Enabling/disabling dependency label editing

To disable editing of the dependency labels, look in client/js/tree_editor_app.js for the call to 
.setDepClickCallback(...)
and comment it out.

2.1.3 Customizing the dependency label set

To customize the available dependency labels, look in client/js/tree_editor_app.js for the line 
<script type='text/javascript' src='js/generic_dep_labels.js'></script>
and update it appropriately.

2.2 Part-of-speech (POS) labels

2.2.1 Enabling/disabling POS visibility

To control whether the part-of-speech tags are visible or not, Look in client/js/tree_editor_app.js for the line with

and set it to

or 
.setShowPos(false)

2.2.2 Enabling/disabling POS visibility

To disable editing of the dependency labels, look in client/js/tree_editor_app.js for the call to

and comment it out.

2.2.3 Customizing the dependency label set

To customize the available dependency labels, look in client/js/tree_editor_app.js for the line 
<script type='text/javascript' src='js/generic_dep_labels.js'></script>
and update it appropriately.

3.0 Guidelines

The annotation guidelines provided in this project may not be appropriate for your annotation project. As such, you will likely wish to replace client/guidelines.html and possibly client/js/guidelines.js with your own custom guidelines.
